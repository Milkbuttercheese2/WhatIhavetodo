# 기능 제안서 — 기능 추가 vs 단순 유지 (조달청 내부망 배포 관점)

> 2026-07-11 작성 (v2.3 기준). 질문: 향후 내부망 조달청 직원들이 사용한다는 것을 고려했을 때
> 기능을 추가하는 게 좋은가, 심플하게 유지하는 게 좋은가? 추가한다면 무엇을 추가할 것인가?
> 방법: 기존 `시장 리서치 보고서.md`(2026-07-10, 유사 서비스 8종 검증 조사)를 근거의 뼈대로 삼고,
> 그 보고서가 공백으로 명시한 항목(국내 공공 현장의 실제 대체재, MS To Do의 내부망 사용 가능성)을 보완 조사했다.

---

## 1. 결론 (권고)

**"심플한 코어 유지 + 검증된 갭 3개만 선별 추가"의 절충안을 권고한다.**
전면적 기능 확장도, 현상 동결도 아니다. 이유는 단순하다:

- 이 앱의 **경쟁력 자체가 단순함**이다 — 설치 없음, 계정 없음, 학습 없음, 칸반 수동 이동 없음(자동 배치).
  기능을 넓게 늘리면 이 정체성이 희석되고, 내부망 보안검토·저사양 PC·다양한 IT 숙련도라는
  공공 환경 제약과 정면으로 충돌한다.
- 그러나 시장조사에서 **복수 앱이 공통으로 제공하는 표준 기능 5가지** 중 이 앱에 없는 것이 있고,
  그중 일부는 조달청 업무 특성(정기·반복 업무, 전화 응대)과 정확히 맞물린다.
  "없어서 아쉬운 표준 기능"만 채우는 것은 단순함을 해치지 않는다.

한 줄 요약: **넓히지 말고, 깊게 채워라.**

## 2. 사용 환경 분석 — 조달청 내부망이 뜻하는 것

| 제약 | 시사점 |
|---|---|
| 망분리 (인터넷 차단) | 서버·계정·동기화·AI API 기반 기능은 원천 배제. 완전 오프라인이 전제조건 |
| 무설치 요구 (관리자 권한 없음) | 단일 포터블 exe 유지가 필수. 런타임·프레임워크 추가 요구 금지 |
| 보안성 검토 | 기능·의존성이 적을수록 반입 승인에 유리. 네트워크 코드 0줄이 강점 |
| 사용자 스펙트럼 | 전 연령·전 직급이 교육 없이 써야 함 → 설정·개념이 늘수록 도입 장벽 |
| 실제 대체재 | 상용 앱이 아니라 **엑셀 서식·수첩·포스트잇** — 공공 현장의 일정/업무관리가 엑셀 서식 중심임은 다운로드 자료 생태계에서 확인됨. 이길 대상은 Todoist가 아니라 엑셀이다 |

**경쟁 앱들이 내부망에 못 들어오는 이유** (= 이 앱의 해자):
- **MS To Do**: Microsoft 계정·클라우드 동기화 필수 — 에어갭 환경에서 지원되는 구성이 아님 (보완 조사에서 확인. 에어갭에서는 오프라인 정품인증용 영구 라이선스 제품군만 지원되며, 클라우드 구독형 앱은 지원 대상이 아님).
- **Todoist/TickTick 등**: 계정 + 서버 동기화 전제. 오프라인 3강(Everdo·Super Productivity·Obsidian)도 설치 또는 런타임 필요.
- **온나라(→온AI)**: 조직 단위 전자결재·문서관리 중심. 개인의 "전화 받고 던져두는 메모"를 다루지 않음 — 비경쟁·보완 관계 (기존 보고서 §1, 단 온AI의 개인 기능 흡수 가능성은 계속 관찰 필요).

즉 조달청 내부망에서 이 앱의 실질 경쟁자는 **엑셀 서식과 수첩**이고,
이들 대비 승부처는 기능 수가 아니라 **"입력이 더 빠르고, 잊어버리지 않게 해준다"** 단 두 가지다.

## 3. 시장조사 요약 (상세는 `시장 리서치 보고서.md`)

- "완전 오프라인·계정 없음" 태스크 앱은 확립된 카테고리다 (Everdo, Super Productivity, Obsidian Tasks — 신뢰도 high). 이 앱은 그 카테고리에서도 가장 극단(에어갭·무설치 단일 exe)에 있다.
- 복수 앱 교차 검증된 **킬러기능 패턴 5가지**: ① 전역 빠른 캡처 단축키 ② 자연어 한 줄 파싱 ③ 반복 일정 ④ 주기적 리뷰 워크플로 ⑤ 저장 필터 뷰. **전부 서버·계정·AI 없이 구현 가능** — 내부망 제약과 충돌하지 않는다.
- 이 앱이 이미 가진 차별점: 마감시각에서 항상 파생되는 자동 배치 보드(수동 이동 없음), 전화 응대 도메인 필드(관련인·식별번호), 캘린더·XLSX 내보내기 전부 무료, v2.3부터 전역 단축키 캡처 + 트레이 상주(패턴 ① 이미 충족).

## 4. 권고안 A — 지켜야 할 단순함 (하지 말 것)

| 하지 말 것 | 이유 |
|---|---|
| 계정·서버·동기화·협업(공유 보드) | 내부망 원천 불가 + 보안검토 리스크 + 온나라 영역 침범 |
| AI 요약·추천 | 오프라인 모델 동봉 시 exe 비대화, 검토 통과 불리, 실효 낮음 |
| 칸반 커스터마이즈(열 추가/수동 이동) | "배치는 항상 파생"이라는 핵심 설계(placeOf)를 파괴. 엑셀과의 차별점 상실 |
| 포모도로·습관 추적 등 번들 확장 | TickTick식 "스위스 아미 나이프" 노선은 이 앱 포지션의 반대말 |
| 설정 항목 증식 | 교육 없이 쓰는 도구라는 전제 훼손. 새 기능은 기본값만으로 동작해야 함 |

## 5. 권고안 B — 추가할 기능 (우선순위)

기존 로드맵(리서치 보고서 §6)을 조달청 관점으로 재평가한 결과다. ③ 전역 캡처는 v2.3에서 이미 반영됐다.

| 순위 | 기능 | 조달청 관점 근거 | 난이도 | 제안 버전 |
|---|---|---|---|---|
| ① | **반복 일정** ("매주 월요일", "매월 25일") | 4개 앱 공통 표준이자 이 앱 최대 갭. 조달 업무는 정기성이 강함 — 월 마감, 정기 보고, 계약 갱신, 정기 점검. 현재는 매번 재입력해야 해서 엑셀 대비 열세인 유일한 지점. "완료 시 다음 due 재계산" 방식이면 자동 배치 모델과 충돌 없음 | 중 | v2.4 |
| ② | **한국어 자연어 한 줄 파싱** ("내일 2시 홍길동 회신") | 전화 응대 중 한 줄 입력이 핵심 시나리오인 이 앱에서 캡처 속도를 직접 올림. Todoist 시그니처 기능의 오프라인 한국어판. 기존 datetime.js 다중 포맷 파서의 자연스러운 확장 | 중 | v2.5 |
| ③ | **주간 리뷰 뷰** (분류대기 장기 방치 + 마감 임박 일괄 점검) | OmniFocus의 검증된 킬러기능. "던져두고 잊는" 앱의 약점(던져두고 영영 안 보는 것)을 스스로 막아줌. 순수 렌더 로직이라 저위험 | 하 | v2.5~ |
| ④ | 조달청 특화 기본값 (식별번호 종류 기본 프리셋: 입찰공고번호·계약번호·요청번호 등) | 신규 기능이 아니라 **기존 id_kinds 기능의 기본값 조정** — 비용 거의 0. 배포 전 실제 사용자에게 명칭 확인만 필요 | 최하 | 배포 시점 |
| ⑤ | 저장 필터 뷰 | 검증된 패턴이나 개인 보드 규모(수십 건)에선 검색으로 충분할 수 있음 — 사용자 요청이 나올 때까지 보류 | 중 | 보류 |

**판단 기준으로 제안하는 3문 테스트** — 앞으로 어떤 기능 요청이 와도:
1. 내부망(오프라인·무설치)에서 완결되는가?
2. 기본값만으로 동작하고, 안 쓰는 사람 눈에는 안 보이는가?
3. "입력이 빨라지거나, 잊는 걸 막아주는가?" (둘 다 아니면 이 앱 일이 아님)

## 6. 리스크와 한계

- **망분리 완화 추세**: N²SF 전환으로 2~3년 뒤 타깃 환경이 좁아질 수 있음 (기존 보고서 §7). 다만 C등급 물리 분리는 존속 — 조달청 내부망은 당분간 유효한 타깃.
- **온AI의 개인 기능 흡수 가능성**: 관찰 필요. 흡수되더라도 "무설치 개인 도구" 포지션은 조직 시스템과 성격이 다름.
- **리서치 공백**: 국내 개인용 앱(네이버 캘린더, 두잇 등)은 검증 통과 자료가 없어 미조사 (기존 보고서 §7 명시). 단, 이들도 전부 계정·온라인 전제라 내부망 결론에는 영향 없음.
- **가장 효과적인 후속 조사는 시장이 아니라 사용자**: 조달청 직원 2~3명에게 "지금 개인 업무를 뭘로 관리하는가"(엑셀? 수첩? 온나라 메모?)를 직접 묻는 것이 어떤 외부 리서치보다 로드맵을 정확하게 만든다. ④(식별번호 기본값)는 이 인터뷰와 함께 확정할 것.
