# 디자인 정책 (UI Design Policy)

이 문서는 '뭐하려 했더라'가 v2.4.5~v2.5.6에 걸쳐 정립한 **화면 디자인 규칙**을 한곳에 모은 것이다.
새 UI를 추가하거나 기존 화면을 고칠 때 이 규칙을 따른다 — 목표는 *깔끔하고 정렬된, 한 시스템처럼 보이는 화면*이다.
모든 값은 `src/styles.css`의 실제 토큰과 일치한다(값을 바꾸면 이 문서도 함께 고친다).

---

## 0. 큰 원칙 (먼저 읽기)

1. **한 시스템처럼 보이게.** 새 요소는 이미 있는 요소의 색·모양·간격·글자 규칙을 그대로 물려받는다.
   새 색·새 크기·새 모서리 곡률을 즉흥적으로 만들지 않는다. 필요하면 아래 토큰을 쓴다.
2. **정렬이 최우선.** 같은 열에 있는 입력칸·라벨·값은 시작선(왼쪽)과 끝선(오른쪽)을 맞춘다.
   드래그 핸들이 없는 줄도 핸들 있는 줄의 입력칸과 들여쓰기를 맞춘다.
3. **한 줄 원칙.** 카드·목록의 각 정보는 한 줄로, 넘치면 줄바꿈이 아니라 **말줄임(…)**.
4. **가로 스크롤 금지.** 어떤 창 폭에서도 요소가 화면 밖으로 나가거나 페이지 가로 스크롤이 생기면 안 된다.
5. **데이터는 안 건드린다.** 보기 방식·배치·표기 변경은 렌더 시 계산만 — 저장 구조/JSON/DB는 불변.

## 1. 색 팔레트 (`:root` 토큰)

색은 전부 CSS 변수다. 팔레트를 바꾸려면 `:root` 토큰만 고치면 전파된다. **하드코딩 색 지양**
(예외: Windows 표준 닫기 hover `#e81123`, 임박 시각칩 배경처럼 의미가 고정된 소수).

| 용도 | 토큰 | 값 |
|---|---|---|
| 본문 글자 / 옅은 글자 | `--ink` / `--ink-soft` | `#1f2a33` / `#68717c` |
| 경계선 / 옅은 경계 | `--line` / `--line-soft` | `#eceef0` / `#f3f4f5` |
| 배경(종이톤) / 카드 | `--paper` / `--card` | `#f4f3f0` / `#ffffff` |
| **주 액센트(청록)** | `--focus` = `--slate` = `--amber` | `#4d938a` |
| 액센트 배경 | `--slate-bg` / `--amber-bg` | `#eef2f0` |
| 위험/삭제 | `--red` / `--red-bg` | `#C0271C` / `#FBE9E7` |
| 완료/비활성 | `--done` | `#6b7787` |
| 삭제 아이콘 기본 | `--icon-muted` | `#aeb4be` |

**업무 단계 색**(컬럼 점 + 카드 좌측 세그먼트 **전용** — 다른 데 쓰지 않는다):

| 단계 | 토큰 | 값 | 의미 |
|---|---|---|---|
| 분류 대기 | `--stage-inbox` | `#8a919b` | 무채 회색 |
| 오늘 처리 | `--stage-today` | `#cf6a55` | 적색 |
| 진행 중 | `--stage-doing` | `#c69a52` | 황토 |
| 예정·대기 | `--stage-planned` | `#5aa08f` | 녹색 |

**태그(배지) 색은 종류별로 겹치지 않게 구분한다:**
- 시각(마감·점검): 청록 `--slate-bg`/`--slate`
- 담당(#담당): 뮤트 블루 `#e3ebf6`/`#3c6c9e`
- 식별번호: 보라 `#e9e4f2`/`#5b4a86`
- 임박 시각(지남~2시간): 코랄 레드 `#fbe3dd`/`#c0392b`

## 2. 타이포그래피

- **서체**: Pretendard 로컬 번들(`fonts/PretendardVariable.woff2`) → 맑은 고딕 폴백. 외부 웹폰트 금지(내부망).
- **크기 정책(v2.5.2)**: 본문 UI 텍스트는 **12.5px**(Pretendard 상속), 카드 메모·세부는 14px,
  섹션 제목·라벨은 11px/600, 태그는 10~11px.
- **모노 폰트(`--font-mono`)는 날짜·시각·번호·카운트 숫자에만.** 일반 텍스트에 쓰지 않는다.
- **⚠️ CSS `font` 축약형에 전역 키워드(`inherit`) 섞지 말 것.** `font:14px/1.5 inherit`처럼 쓰면
  선언 전체가 무효가 되어 브라우저 기본 폼 폰트로 렌더된다(v2.5.1 빠른 메모창 버그). 반드시
  `font-family`/`font-size`/`line-height` **롱핸드로 분리**.
- **`<button>`은 폰트를 상속하지 않는다** → 버튼에는 `font-family:inherit`을 명시(옵션 카드·툴 버튼 등).

## 3. 카드 (보드/완료)

- **모양**: 흰 배경, `border-radius:13px`, 1px 경계, hover 시 살짝 떠오르는 소프트 섀도.
- **단계 표시**: 좌측 **짧은 색 세그먼트**(`.card::before`, 4×20px) — 풀 높이 바 아님.
- **체크박스**: 원형(`.chk`, 18px, `border-radius:50%`), 켜지면 청록 배경 + 흰 체크.
- **카드 3줄 원칙(v2.5.2)** — 각 줄 1행 말줄임:
  1. **메모** 1줄
  2. **세부 할 일** 1줄 (▸ 접두, 있으면)
  3. **시각** 1줄 (`마감:` 또는 `점검:` — 마감보다 앞서는 미완료 세부가 있으면 `점검:`만, 없으면 `마감:`만)
  - 진행상황(#세부 x/n)·담당(#담당)은 카드에서 뺀다 → 자세한 건 카드를 열어 양식에서.
  - 완료 탭 카드 = **메모 + `완료: 날짜 시각`** 만.
- **시각 태그**:
  - 표기: `마감: 7/18(토) 14:22` — **남은 시간 꼬리표(N분후/N시간후) 없음**.
  - 긴급도 색: **임박(지남~2시간 이내)만 코랄 레드 강조, 나머지는 전부 중립.** (5단계 색은 폐지 — 어지러움)
  - 알람 점(`.adot`) 3단계: ○ 대기 / ● 울림(빨강) / ● 확인함(초록). 완료 업무는 점 없음.
- **주기 배지**: 시각 줄에 인라인, 10px로 축소(`flex:none` — 좁은 열에서도 시각과 한 줄에).

## 4. 태그·배지 규칙

- 알약형(`border-radius:999px`), 모노 폰트, 10~11px, `#키:값` 꼴로 통일(`#담당:`, `마감:`, `점검:`).
- 한 줄 유지가 우선 → 메타 줄은 `flex-wrap:nowrap;overflow:hidden`, 태그는 말줄임.
- 종류별 배경색은 **§1의 태그 색 표**를 따른다(겹치지 않게).

## 5. 양식·모달 — 정렬 규칙 (핵심)

### 5.1 행(row) 구조와 드래그 정렬
- **순서 바꿀 수 있는 행에는 드래그 핸들 `⠿`**(`.drag-handle`)을 왼쪽에 둔다.
  현재 적용: 세부 할 일 / 식별정보 / 관련인. (`enableDragReorder(container, rowSel, '.drag-handle')`)
- **Enter로 다음 줄 추가/이동**: 마지막 행이면 새 행 추가, 아니면 아래 행의 **같은 칸**으로 포커스.
  한글 IME 조합 중 Enter는 무시(`e.isComposing||e.keyCode===229`). 적용: 세부 할 일 / 관련인.

### 5.2 들여쓰기 정렬
- 드래그 핸들 폭 = **⠿ 11px + 간격 7px = 18px**. 핸들 없는 줄(접수·마감시각 등)은 **입력칸을
  18px 들여써** 핸들 있는 줄의 첫 칸과 시작선을 맞춘다(라벨 글자는 왼쪽 끝 유지).
- 체크박스까지 있는 줄(세부 할 일)의 제목칸 시작선 = **⠿11+7+체크18+7 = 43px**. 이 줄의 둘째 줄
  요소는 `margin-left:43px`로 제목칸과 정렬.

### 5.3 섹션 제목
- 공용 규칙 하나로: **`.fm-inner .t-head` (11px / 600 / `--ink-soft`)**. 섹션마다 개별 규칙을 만들지
  않는다(파일 링크 제목만 빠져 크게 렌더되던 v2.5.3 버그의 교훈 — 개별 스코프 규칙은 누락 위험).

### 5.4 양식 항목 순서(v2.5.3)
메모 → 세부 할 일 → 접수·마감시각 → 식별정보 → 관련소속·관련인·연락처 → 파일 링크.

### 5.5 표준 모달 구조
모든 팝업은 같은 문법을 쓴다(예: 보드 모드 선택 / 주기 업무 / 프리셋):
```
<div class="modal-bg" id="...">
  <div class="modal">
    <h3>제목</h3>
    <div class="m-sub">한 줄 설명</div>
    …본문(옵션 카드·입력·목록)…
    <div class="m-actions"><button class="btn btn-ok" id="...Close">닫기</button></div>
  </div>
</div>
```
- **body 직속으로 append**(`document.body.appendChild`)해 어느 탭에서든 뜨게.
- **ESC 닫기**(main.js F14 핸들러에 추가) + **배경 클릭 닫기**(`e.target.id===모달id`).
- **선택형 옵션은 카드(`.bm-opt` 류)**: 둥근 카드, 선택 시 청록 경계 + `--slate-bg` 배경 + 청록 제목.
  탭/필과 같은 시각 언어(둥근, 청록 `.on`). 클릭 즉시 적용하고 표시를 갱신(모달은 열어둠).

## 6. 반응형 정책 (v2.5.4~2.5.6)

- **창은 두 상태만**: **560px 컴팩트** 또는 **전체화면**. 자유 리사이즈 없음(`resizable:false`),
  타이틀바 □ 버튼으로 전환. 애매한 중간 폭 레이아웃은 만들지 않는다.
- **`@media(max-width:1080px)`**: 모든 주요 컨테이너를 560px 폭으로 고정·가운데 정렬, 보드는 단일 열,
  헤더 컴팩트(부제 숨김·시계 오른쪽), 입력부 세로 스택.
- **560px에서 넘치기 쉬운 곳은 접거나 내린다**(이 미디어 블록 안에서 처리):
  - 접수·마감시각 그리드 → 1열
  - 세부 할 일 → **2줄**(첫 줄 제목+체크+삭제 / 둘째 줄 담당·날짜·시각). `order`만으로는 폭이 남으면
    한 줄에 다 들어가므로, order 0 그룹 뒤 `::after{flex:0 0 100%;height:0}` **강제 줄바꿈**을 넣는다.
  - 프리셋 새 입력칸 → 1열
- **새 화면을 추가하면 반드시 560px·480px에서 오버플로 감사**(모든 뷰/모달 순회, 요소 `right>W` 및
  `scrollWidth>innerWidth` 검사)를 돌려 확인한다.

## 7. UI 오버플로 방지 기법 (정책 — 반드시 지킬 것)

> **근본 원인부터.** 리눅스 개발 렌더(Playwright/Chromium)와 **Windows 배포 렌더(WebView2)**
> 는 **폰트 metric·네이티브 컨트롤(날짜/시각 피커·select·스크롤바) 크기가 미묘하게 다르다.**
> 특히 Pretendard 로드 실패 시 폴백(맑은 고딕)으로 글자 advance-width가 바뀌고, 세로
> 스크롤바가 **Windows에선 폭을 차지**(리눅스 오버레이는 0)한다. 그래서 **한 플랫폼 픽셀에
> 딱 맞춘 하드코딩은 다른 플랫폼에서 반드시 터진다.** 아래는 "픽셀 튜닝" 대신 **폭이 알아서
> 줄어드는 CSS**로 짜는 규칙이다.

**A. flex/grid 자식은 `min-width:0`** — flex 자식 기본값 `min-width:auto`는 내용보다 작게
  안 줄어들어 넘침의 최다 원인. 텍스트를 담는 카드 본문·양식 칸·세부 제목·dt 입력은 전부
  `min-width:0`(또는 grid는 `minmax(0,1fr)`). 이미 `.card-body`·`.fm-field`·`.fsub-title`
  ·`.dt-date` 등에 적용돼 있으니 새 칸도 동일하게.

**B. 긴 텍스트는 폭을 밀지 못하게 자른다** — 한 줄은 `overflow:hidden;text-overflow:ellipsis;
  white-space:nowrap`, 여러 줄은 `-webkit-line-clamp`(메모 2줄). 텍스트가 컨테이너를 넓히는 일
  자체를 없앤다.

**C. grid 열은 `1fr` 대신 `minmax(0,1fr)` / `max-content`** — 순수 `1fr`(=`minmax(auto,1fr)`)
  은 내용 이하로 안 줄어 넘친다. 줄어야 하는 열은 `minmax(0,1fr)`, 내용폭 고정 열(날짜·시각처럼
  피커가 필요한 칸)은 `max-content`. (v2.5.7 `fm-grid`가 `repeat(4,1fr)`→`repeat(2,max-content)`
  로 바뀐 이유.)

**D. 세로 스크롤바 자리 상시 확보** — `html{scrollbar-gutter:stable}`. 목록이 줄어 스크롤바가
  사라질 때 폭이 11px 늘며 `margin-left:auto` 요소가 옆으로 튀는 현상(Windows 클래식 스크롤바
  전용)을 원천 차단. (v2.5.8)

**E. 가로 스크롤 하드 차단** — `body{overflow-x:clip}`(v2.5.4). 어떤 실수도 가로 스크롤바로
  새어나오지 않게.

**F. 단일 열에선 `margin-left:auto`(우측 고정) 대신 전체 폭** — 우측 고정 요소는 빈 띠에 떠
  어색하고 스크롤바 토글에 흔들린다. `@media(max-width:1080px)`에서 검색창 등은 `width:100%`.
  (v2.5.8)

**G. 좁은 폭 강제 줄바꿈** — `order`만으론 폭이 남을 때 한 줄로 붙는다. order 0 그룹 뒤
  `::after{content:"";flex:0 0 100%;height:0}`로 항상 다음 줄로 내린다(세부 할 일 2줄).

**H. 고정폭 요소는 `flex:none`, 옆 텍스트는 `flex:1;min-width:0`** — 아이콘·배지·체크박스·
  마스코트가 눌리지 않고, 텍스트만 줄어 잘리게.

**I. 네이티브 컨트롤 폭에 컨테이너를 딱 맞추지 말 것** — 날짜/시각 피커·select는 Windows에서
  더 넓다. 칸을 그 폭에 정확히 맞추면 Windows에서 넘친다. 여유를 두거나 `flex:1;min-width:0`로.

**J. 검증은 리눅스가 아니라 실제 Windows에서** — 픽셀을 리눅스 렌더로 맞추지 말고,
  `tools/win-render.mjs`(windows-latest Edge = WebView2 엔진·Windows 폰트)를 돌려 **560px·전체화면
  에서 `scrollWidth <= clientWidth`를 단언**한다. 새 화면/모달을 추가하면 이 검사를 통과시킨 뒤
  머지. (`.github/workflows/win-render.yml` 수동 트리거 → 오버플로 시 CI 실패.)

## 8. 기타 요소

## 9. 절대 하지 말 것 (요약)

- 새 색/새 곡률/새 글자크기 즉흥 생성 (토큰·기존 값 재사용).
- `font` 축약형 + `inherit` 혼용, `<button>`에 `font-family` 생략.
- 카드/목록 정보를 두 줄로 흘리기 (말줄임으로).
- 어떤 폭에서든 가로 스크롤·요소 화면 밖 이탈.
- **한 플랫폼(리눅스) 렌더에 픽셀을 하드코딩** — 폭은 §7의 intrinsic-sizing(min-width:0·
  minmax(0,…)·flex·말줄임)으로 짜고, 실제 Windows(§7-J)에서 검증한다.
- 표시 명칭/배치를 바꾸며 저장 키·스키마·JSON 포맷 건드리기.
- 섹션마다 제목 스타일 개별 정의 (공용 `.t-head` 사용).
