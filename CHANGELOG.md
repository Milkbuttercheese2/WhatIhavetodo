# 변경 이력 (CHANGELOG)

깃허브에 업데이트할 때마다 무엇이 바뀌었는지 이 파일에 기록한다.
버전 규칙: 표준 X.Y.Z(semver) — X=큰 개편, Y=기능 추가, Z=버그 수정·사소한 조정.
**정책(소유자 지정): 사소한 수정이라도 배포(빌드/머지)할 때마다 패치(Z)를 올린다(v2.4.0 → v2.4.1 → …).**
(v2.31까지의 옛 규칙: 큰 업데이트 +0.1, 작은 업데이트 +0.01, 십진수 크기순 —
v2.2 → v2.21 → … → v2.3(=2.30) → v2.31. 즉 v2.3은 v2.23 다음(더 최신) 버전이다.)

> **⚠️ 버전 번호 안내(꼭 읽기):** 아래 v3.0.0·v3.1.0 항목은 *다른 계정에서 진행되던 미배포
> 개편 작업분*이며 **실제로 배포된 적이 없다.** 당시 파일 링크·이름 변경 정도의 변화를 두고
> 대규모 개편도 아닌데 버전을 3.0.0으로 **임의 상향(잘못된 범프)** 한 것이라, 소유자 결정으로
> 바로잡아 **구 v2.31 라인을 잇는 v2.4.0**으로 정식 배포한다. 즉 v3.x는 내부 이력일 뿐 실 배포
> 버전은 **v2.4.x(현재 v2.5.0)**이고, 다음은 v2.4.x/v2.5.0으로 이어간다(진짜 대규모 개편 전엔 3.x로 올리지 말 것).
> (자동 업데이터·버전 비교 로직이 없어 번호 역전은 무해하다.)

### 수정 (데이터 안전 — 코드 감사 반영)
- **자동 백업 복구/정리의 파일명 정렬 오류(데이터손실)**: `wmhh_`·`wmhh_prerestore_`·
  `wmhh_preimport_` 접두사가 섞이면 파일명 문자순 정렬에서 `'p' > '2'`라 오래된 pre* 스냅샷이
  항상 최신으로 뽑혀, DB 손상 복구가 옛 데이터로 되돌리고 정리가 최신 백업을 지웠다. 파일명에
  박힌 타임스탬프(`YYYYMMDD_HHMMSS`)로 정렬하도록 수정(`stamp_key`) + 회귀 테스트 추가.
- **레거시/부분 백업 복원 거부**: `RecurDef`/`RecurTime`에 `#[serde(default)]`가 없어 `freq`/`time`
  누락된 정기 정의 한 건이 전체 복원을 실패시키던 문제(정기함은 제거됐고 이 데이터는 보존용).
- **손상된 알람 상태 디코드 방향**: 알 수 없는 알람 TEXT를 `Fired(true)`(확인함)로 읽어 알람이
  다시 안 울리던 것을 `Fired(false)`(미확인)로 — 안전방향은 '울리게'.
- **빠른 메모 등록 직후 크래시 시 중복 등록**: 등록과 동시에 초안(captureDraft)을 즉시 비워
  다음 실행에 같은 메모가 한 번 더 등록되던 문제 방지.
- **JSON 백업에 임시 초안 포함**: 백업 시 `captureDraft`를 제외해 복원 후 유령 항목 생성 방지.
- **검색어 오염**: `filters.js`의 식별정보·세부 제목에 `||''` 가드 추가(누락 필드가 "undefined"로
  검색되던 것 방지).
- **저장 완료 대기**: `STORE.saveAll`이 진행 중 프로미스를 반환해 `await`가 실제 저장 완료를 기다리게.
- **비-Windows 컴파일**: Windows 전용 `set_overlay_icon`을 `#[cfg(windows)]`로 게이트해
  리눅스/맥에서도 crate가 빌드·테스트되게(Windows 동작 불변).

### 수정
- **바로 입력창 세로 확장 버그**: WebView2에서 드래그 리사이즈가 잘 안 잡히던 문제를 근본 해결 —
  내용을 입력하면 입력창이 **자동으로 세로로 늘어난다**(최소 높이 유지, 최대 화면 60%까지 늘고 그 뒤
  스크롤). 드래그 리사이즈 핸들도 그대로 둠.
- **좁은 화면에서 요소가 밖으로 튀어나가던 문제**: 헤더의 모드 토글(시간·담당자)이 창을 줄이면
  화면 밖으로 잘리던 것을 수정 — 좁은 창에선 부제(설명글)를 숨기고 브랜드 줄바꿈을 허용해 폭 안에
  들어오도록(줄 수를 늘리지 않고 가로폭을 줄이는 방향). **현재시각(시계)도 헤더가 접히는 폭부터
  항상 오른쪽 끝에 정렬**되도록(넓은 창 배치는 그대로).
