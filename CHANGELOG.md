# 변경 이력 (CHANGELOG)

깃허브에 업데이트할 때마다 무엇이 바뀌었는지 이 파일에 기록한다.
버전 규칙: 표준 X.Y.Z(semver) — X=큰 개편, Y=기능 추가, Z=버그 수정·사소한 조정.
**정책(소유자 지정): 사소한 수정이라도 배포(빌드/머지)할 때마다 패치(Z)를 올린다(v2.4.0 → v2.4.1 → …).**
(v2.31까지의 옛 규칙: 큰 업데이트 +0.1, 작은 업데이트 +0.01, 십진수 크기순 —
v2.2 → v2.21 → … → v2.3(=2.30) → v2.31. 즉 v2.3은 v2.23 다음(더 최신) 버전이다.)

> **⚠️ 버전 번호 안내(꼭 읽기):** 아래 v3.0.0·v3.1.0 항목은 *다른 계정에서 진행되던 미배포
> 개편 작업분*이며 **실제로 배포된 적이 없다.** 당시 파일 링크·이름 변경 정도의 변화를 두고
> 대규모 개편도 아닌데 버전을 3.0.0으로 **임의 상향(잘못된 범프)** 한 것이라, 소유자 결정으로
> 바로잡아 **구 v2.31 라인을 잇는 v2.4.0**으로 정식 배포한다. 즉 v3.x는 내부 이력일 뿐 실 배포
> 버전은 **v2.4.x(현재 v2.4.3)**이고, 다음은 v2.4.x/v2.5.0으로 이어간다(진짜 대규모 개편 전엔 3.x로 올리지 말 것).
> (자동 업데이터·버전 비교 로직이 없어 번호 역전은 무해하다.)

## v2.4.4 — 2026-07-15

### 변경
- **알람 점을 "울림 여부" 3단계로 단순화.** 기존 4단계(○ 대기 · ● 울림 · ● 확인 · ● 미룸)에서
  '미룸(파랑)' 색 구분을 없애 **○ 대기(안 울림) · ● 울림(빨강, 시각 지남) · ● 확인함(초록)** 세 가지로
  줄였다. 스누즈(미룸)한 알람도 이미 울린 것이므로 '울림(빨강)'으로 함께 표시한다. (완료 업무·세부는
  점을 표시하지 않는 규칙은 그대로.)
- **세부 점검이 여러 개일 때 '울린 것(지난 미완료 점검)'을 카드에 우선 표시.** 이전에는 가장 임박한
  '미래' 점검을 먼저 보여줘서, 한참 전에 울린 점검이 미래 점검에 가려 카드 앞면에 안 보이던 문제가
  있었다. 이제 지나서 울린 점검이 있으면 그것(가장 오래된 것)을 먼저 보여줘 빨간 점이 바로 눈에 띈다.
- **주기 업무에서 '며칠마다'(N일 간격) 입력을 제거.** 이제 주기 유형은 **매주 요일 / 매월 며칠** 두
  가지다. (이미 저장된 '며칠마다' 정의는 계속 생성·표시되어 데이터 호환은 유지 — 신규 입력에서만 빠짐.)

## v2.4.0 — 2026-07-14

'다른 계정'에서 진행되던 개편(구 v3.0.0·v3.1.0)을 정식 배포용으로 통합·정리한 버전.
기존 데이터·JSON 백업·DB 백업과 완전 호환(내부 식별자·스키마 유지 — 데이터 경로가 끊기지 않음).

### 추가
- **주기 업무 — 부모-자식 자동 생성 구조(정규화)**: **[설정] 메뉴 → 주기 업무 입력·관리**
  에서 *공통 내용(제목/메모)과 반복 주기만* 최소로 등록하는 '부모 정의'를 만든다. 예정일이 되면
  그 내용으로 **자식 업무가 보드에 자동 생성**되고, 자식은 일반 업무처럼 완료하면 [완료] 탭으로
  간다. 자식은 `recurId`로 부모를 가리키고(자식→부모), 부모는 자식 목록을 역참조한다(부모→자식).
  반복 주기 3종: **매주 요일 / 매월 며칠(말일 자동보정) / 며칠마다**. 같은 화면에서
  목록·수정·일시정지·삭제·다음 예정·생성 건수를 볼 수 있다. 카드에는 '주기' 배지 표시.
- **자식 생성은 매주 월요일 오전 6시 배치**: 월요일 06:00을 지나면 그 주(월 06:00~다음 월 06:00)에
  해당하는 회차를 한꺼번에 생성한다. 자식의 마감시각은 원래 회차 시각 그대로라, 금요일 회차도
  월요일 아침에 미리 예정·대기로 올라온다. (오래 실행하지 않았어도 14일 이내 밀린 회차는 따라잡아 생성.)
- 매월 반복에서 그 달에 없는 날짜(예: 31일)는 그 달 말일로 자동 보정.
- 주기 업무 **시각 입력은 콜론 없이도** 가능(`0930`·`930` → `09:30`으로 자동 정규화).

### 변경
- **검색 속도 개선을 위해 Everything(voidtools) 연동 전부 제거**: 보드 검색·빠른 메모창의
  파일 검색, 설정의 자동 실행/사용 토글, Rust 커맨드(`everything_search` 등)·자동 실행 로직까지
  모두 삭제. 파일 링크(카드에서 파일 열기/폴더 열기) 기능 자체는 그대로 유지.
- **빠른 메모/빠른 검색 창 UI를 메인 앱과 통일**: 폰트(Pretendard/맑은 고딕)·글씨크기·색을
  메인 앱 팔레트(종이 톤·잉크색)로 맞춘 밝은 플로팅 카드로 정리(기존 어두운 스포트라이트풍 →
  앱과 일관된 밝은 톤). 검색 결과는 '내 업무' 단일 열.
- **빠른 메모창 검색(quick_search) 최적화**: 기존 `LEFT JOIN` 4개 + `DISTINCT`는 한 업무에
  관련행(세부·관련인·식별번호·파일)이 많을수록 중간 결과가 폭증한 뒤 다시 접어야 해서 데이터가
  쌓일수록 급격히 느려졌다. 이를 상관 `EXISTS` 서브쿼리로 바꿔 각 업무를 한 번만 훑고, 자식
  검색은 기존 `item_id` 인덱스를 그대로 탄다(결과·정렬·저장 형식 동일, 스키마 변경 없음).
- **주기 업무를 기존 양식 입력에서 분리**: 양식 입력 창에는 더 이상 반복 설정이 없다(주기는
  전용 [주기 입력]으로). 예전의 '완료 시 자기 자신 재장전' 방식은 부모-자식 생성 방식으로 대체.
- **캘린더는 '진행 중'만 표시**: 완료된 업무·완료된 세부 할일은 캘린더에 나타나지 않는다.
- **'마감 지남' 표시 제거**: 상단 '기한 지남 N' 카운터와 카드의 '지남' 강조를 없앴다.
  임박 안내(남은 시간)는 **2시간 이내**만 표시.
- **빠른 메모 팝업**: 그림자 효과 제거 후 더 세련되게 정리(은은한 그라데이션·강조 점·통일된
  타이포). 검색 모드는 '내 업무' 단일 열(파일 열 제거).
- 날짜 선택 **📅 아이콘** 정상 표시(이모지 폰트 폴백 지정).
- 배포 버전 표기 **v2.4.0** (세 매니페스트: package.json·Cargo.toml·tauri.conf.json).

### 내부
- 새 구조: `recur.js`(순수 반복 계산 + `spawnDueOccurrences` 생성기), `recur-box.js`(주기
  입력·관리 UI + 생성 트리거). 부모=`it.recur` 있는 보드 밖 아이템, 자식=`it.recurId`.

- **캘린더 표시 규칙 단순화**: 그날이 **마감일인 업무** + 그날 시각이 걸린 **세부 할일**만 표시.
  ("시각 있는 세부가 있으면 업무 마감일을 숨김" 로직 제거 — 이제 마감일은 항상 뜬다.)

### 추가
- **파일 링크**: 양식에서 업무에 파일 경로를 연결 ([＋ 파일 찾아보기] 또는 경로 붙여넣기).
  카드의 📄 클릭 = 파일 열기, 📂 = 폴더에서 보기. 검색창에서 파일명으로도 카드가 검색되고,
  XLSX 내보내기에 '파일링크' 열 추가. DB에 item_files 테이블 신설(마이그레이션 003),
  JSON 백업에 items[].files로 포함 (구버전 백업은 자동으로 빈 값 처리 — 왕복 호환).
- **Everything(voidtools) 파일 동시검색 (선택 기능)**: Everything의 HTTP 서버 옵션이 켜져
  있으면 보드 검색 시 내 PC 파일도 함께 검색되어 검색창 아래 표시, 클릭으로 바로 열기.
  통신은 127.0.0.1 로컬 한정. Everything 미실행 시 UI가 아예 나타나지 않는 무침습 설계
  (연결 실패 후 2분간 재시도 안 함). 사용법은 `사용 설명서.md` §5 참고.

### 수정
- **드래그 순서변경 미작동 수정**: 세부 할일·프리셋·식별번호 명칭의 ⠿ 드래그가 Windows에서
  아예 시작되지 않던 문제. 원인은 JS가 아니라 Tauri 창 설정 — WebView2에서 기본값
  `dragDropEnabled:true`가 OS 파일드롭을 위해 드래그 이벤트를 가로채 HTML5 드래그를 차단.
  `dragDropEnabled:false`로 해결 (트레이드오프: 창으로의 OS 파일 끌어놓기는 사용 불가 —
  이 앱은 원래 안 씀).
- **세부 할일 전부 완료 시 '예정 · 대기'로 떨어지던 배치 수정**: 이제 세부를 하나라도
  완료한 업무는 전부 완료했어도 (상위 완료 체크 전까지) '진행 중'에 남는다 —
  '예정 · 대기'는 손 안 댄 업무 전용.

  서버 대상 단위테스트 4건 포함). 테스트: JS 91건 + Rust 21건 전부 통과.

## v2.3 — 2026-07-11

v2.2 이후의 작은 중간 업데이트(v2.21, v2.22, v2.23)를 묶어 정식 반영한 **큰 업데이트**.
(v2.23의 다음 버전이 v2.3이다 — 위 버전 순서 규칙 참고.)

### 변경
- **보드 정렬 통일**: 세부할일(점검 시각)과 할일(마감시각)의 레벨 구분을 없애고,
  둘 중 먼저 도래하는 시각 순(오름차순)으로 카드를 정렬. 시각이 없는 카드는 뒤로,
  그 안에서는 최신 등록 순.
- **시각 도래 빨간 카드 제거**: 마감이 지났거나 임박했을 때 카드 전체 배경을
  빨갛게 칠하던 표시(urgent)를 삭제. 마감 태그(pill)의 빨간 지남/임박 표시는 유지.
- 프론트엔드 구조 개편(v2.21): 단일 `app.js`를 관심사별 ES 모듈로 분리
  (`state/store/render/form/placement/datetime/calendar/alarms/backup/presets/dom-utils/main`).
- 도메인 봉합선 정리: 아이템 생성을 `makeItem()` 팩토리로, 완료 토글을 `toggleDone()`
  도메인 연산으로 통일하고, 검색 술어를 `filters.js`로 분리 (동작 변화 없음 —
  반복 일정 등 후속 기능이 얹힐 기반).
