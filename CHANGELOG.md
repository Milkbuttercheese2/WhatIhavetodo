# 변경 이력 (CHANGELOG)

깃허브에 업데이트할 때마다 무엇이 바뀌었는지 이 파일에 기록한다.
버전 규칙: 큰 업데이트 +0.1, 작은 업데이트 +0.01 (`CLAUDE.md` 버전 규칙 참고).
버전은 **십진수 크기순**으로 읽는다: v2.2 → v2.21 → v2.22 → v2.23 → **v2.3**(=2.30) → v2.31 → … → v2.4.
즉 v2.3은 v2.23 **다음**(더 최신) 버전이다.

## v2.4 — 2026-07-11

설정을 아예 못 바꾸게 된 v2.31 이후 상태를 바로잡는 업데이트. 헤더 오른쪽에 **⚙ 설정**
버튼 하나를 두고, 좌측 탭(일반/알림/단축키/데이터)으로 나뉜 **설정 창**을 새로 만들었다.
v2.31에서 제거했던 설정을 되살리고, 흩어져 있던 데이터 버튼을 한곳으로 모으고, 다크 테마와
알람 소리 토글을 추가했다.

### 추가
- **⚙ 설정 창(좌측 탭 사이드바)**: 일반 / 알림 / 단축키 / 데이터 4개 탭.
- **다크 / 라이트 / 시스템 테마**: '시스템'은 OS 설정을 따라가며, OS 테마가 바뀌면 즉시 반영.
  (일반 탭 → 테마)
- **알람 소리 켜기/끄기**: 알림은 그대로 두고 비프음만 끌 수 있다. (알림 탭)
- **빠른 캡처 창 리디자인(macOS Spotlight 풍)**: 투명 창 위에 떠 있는 둥근 카드로 바꾸고
  펜 아이콘·`Ctrl ↵` 키 힌트를 넣었다. 라이트/다크 두 테마를 지원하며, 앱 테마 설정(일반 탭)을
  그대로 따라간다(창이 뜰 때마다 메인 창과 테마를 주고받아 동기화 — 미설정 시 OS 선호를 따름).
  여러 줄을 입력하면 창이 아래로 **자동 확장**되어 윗줄이 가려지지 않는다(최대 7줄, 초과 시 내부 스크롤).

### 변경
- **데이터 버튼 통합**: 헤더에 흩어져 있던 저장 위치 변경·JSON 백업·JSON·DB 불러오기·XLSX
  다운로드를 설정 창 '데이터' 탭으로 모았다. 헤더에는 🔔 알람 토글과 ⚙ 설정만 남는다.
- 🔔 알람 토글은 헤더에 그대로 두되, 설정 '알림' 탭의 '알람 켜기' 체크박스와 상태를 동기화.

### 복원
- **자동 실행·트레이 상주·빠른 캡처 단축키 설정을 되살림**: v2.31에서 제거했던
  Windows 시작 시 자동 실행, 자동 실행 시 트레이로만 시작, 창 X = 트레이 상주,
  전역 캡처 단축키 재지정(라이브 리바인딩)을 '일반'·'단축키' 탭에서 다시 설정할 수 있다.
  (Rust `set_autostart`·`set_capture_shortcut` 커맨드와 시작 시 단축키를 설정값에서 읽는
  경로를 복원. 자동 실행/트레이 상주 설정값은 이미 백엔드가 읽고 있었고 쓰기 경로만 없었다.)

## v2.32 — 2026-07-11

고정폭(1460px) 데스크톱 전용 레이아웃을 **가변폭 반응형**으로 전환 — 창을 좁혀도
가로 스크롤 대신 보드가 자연스럽게 재배치된다. 동작·데이터 변화 없는 순수 레이아웃 개선.

### 변경
- **밴드 컨테이너 유동화**: `body{min-width:1460px}` 제거(→ 320px), 헤더·입력부·요약
  스트립·보드를 `max-width:var(--content-max=1440px); margin-inline:auto`로 통일. 넓은
  화면은 기존과 동일, 좁아지면 폭만 줄어든다. 헤더는 `flex-wrap`으로 버튼이 줄바꿈.
- **보드 4열 가변화**: `repeat(4,365px)` 고정 → `repeat(4,minmax(0,1fr))`, `.col` 고정폭
  해제. 긴 카드가 컬럼을 밀어내지 않도록 `min-width:0` 방어는 유지.
- **콘텐츠 기준 브레이크포인트 추가**: 폭 ≤1024px → 보드 2열(2×2), ≤680px → 1열 + 입력
  행 세로 배치, ≤480px → 폼 그리드 1열. 레이아웃 사이징 토큰(`--content-max/--gutter/
  --col-min`) 도입.
- **좁은 폭 마감 폴리시**: 헤더 툴바(`.head-right`)를 `flex-wrap`으로 감싸 좁은 창에서
  버튼이 잘리지 않고 여러 줄로 접히게, 캘린더 헤더(`.cal-head`)도 `flex-wrap` +
  제목 `white-space:nowrap`로 "YYYY년 M월"이 찌부러지지 않게 정리.

## v2.31 — 2026-07-11

툴바·기능을 덜어내는 정리 업데이트 — "단축키 → 메모창 → Ctrl+Enter 등록" 흐름만 남기고
주변 부가 기능(정기함·단축키 설정·상주 토글·저장 실패 표시)을 제거했다.

### 변경
- **미니 캡처(메모창) 단축키를 Ctrl+Alt+Space 고정으로**: 단축키 변경 UI(⚡ 빠른 메모
  버튼·설정 모달)를 삭제. 다른 프로그램 사용 중 단축키를 누르면 메모창이 뜨고,
  입력 후 Ctrl+Enter로 '분류 대기'에 등록되는 동작 자체는 그대로 유지.
- 트레이 메뉴를 '열기 / 종료' 둘로 정리 ('빠른 메모' 항목 삭제 — 메모창은 단축키로만).
- 창 X = 트레이로 숨김 동작은 유지 (토글 UI만 제거, 기본값 켬).

### 삭제
- **정기함(반복 일정) 기능 제거**: 🔁 정기함 버튼·모달·생성기(reconcileRecur)·카드 🔁 배지
  일괄 삭제. 이미 보드에 올라온 회차는 일반 카드라 그대로 남는다. DB의 recur_defs
  테이블과 JSON 백업의 recurDefs 항목은 **데이터 보존을 위해 왕복만 유지**(스키마
  마이그레이션은 append-only 원칙상 되돌리지 않음) — 기능만 사라졌고 데이터는 유실되지 않음.
- 헤더의 '⚠ 저장 실패' 상태 표시(saveStatus/setStatus) 제거 — 초기 로드 실패 시의
  안내(alert + 저장 차단)는 그대로 유지.
- 설정 모달에 있던 '윈도우 시작 시 자동 실행' 토글 제거 (기존에 켜둔 PC는 그대로 동작).

### 내부
- 프론트 모듈 `recur-box.js` 삭제, `capture-bridge.js`는 메모 라우팅+트레이 안내만 남김.
- Rust 커맨드 `save_recur_defs`/`set_capture_shortcut`/`set_autostart` 제거
  (백업 복원 경로의 recur_defs 저장은 유지).
- 테스트 JS 110 → 88개 (제거 기능의 테스트 삭제), Rust 17개 유지. `cargo check` 경고 0.
- 버전 v2.31 (매니페스트 2.31.0 ×3 + 락파일).

### 산출물
- `최종 프로그램 산출물/뭐해야 했더라.exe`를 v2.31 빌드로 갱신 (build-windows-exe
  워크플로 run #3 — windows-latest MSVC, 테스트 JS 88·Rust 17 통과 후 빌드, 단일 exe 12.5MB).

## v2.3 — 2026-07-11

v2.2 이후의 작은 중간 업데이트(v2.21, v2.22, v2.23)를 묶어 정식 반영한 **큰 업데이트**.
(v2.23의 다음 버전이 v2.3이다 — 위 버전 순서 규칙 참고.)

### 변경
- **보드 정렬 통일**: 세부할일(점검 시각)과 할일(마감시각)의 레벨 구분을 없애고,
  둘 중 먼저 도래하는 시각 순(오름차순)으로 카드를 정렬. 시각이 없는 카드는 뒤로,
  그 안에서는 최신 등록 순.
- **시각 도래 빨간 카드 제거**: 마감이 지났거나 임박했을 때 카드 전체 배경을
  빨갛게 칠하던 표시(urgent)를 삭제. 마감 태그(pill)의 빨간 지남/임박 표시는 유지.
- 프론트엔드 구조 개편(v2.21): 단일 `app.js`를 관심사별 ES 모듈로 분리
  (`state/store/render/form/placement/datetime/calendar/alarms/backup/presets/dom-utils/main`).
- 도메인 봉합선 정리: 아이템 생성을 `makeItem()` 팩토리로, 완료 토글을 `toggleDone()`
  도메인 연산으로 통일하고, 검색 술어를 `filters.js`로 분리 (동작 변화 없음 —
  반복 일정 등 후속 기능이 얹힐 기반).

### 추가
- **정기함(반복 일정)**: 매일/매주(요일 지정)/매월 반복 업무를 정기함(🔁)에 등록하면
  도래한 날에 일반 카드로 보드에 자동 등장(스폰). 반복 규칙은 아이템 속성이 아니라
  보드 밖 생성기(recur_defs)로 분리 — 체크=완료로 떠나는 보드 규칙과 `placeOf()` 불변.
  DB 마이그레이션 002(recur_defs 테이블 + items.recur_id), JSON 백업 자동 왕복.
- **전역 단축키 미니 캡처 창**(v2.23): 다른 프로그램 사용 중에도 단축키로 빠른 메모 입력.
- **트레이 상주 + 자동 시작**(v2.23): 창을 닫아도 트레이에 남고, 윈도우 시작 시 자동 실행 옵션.
- JS 테스트 하네스: node:test + jsdom 기반 자동 테스트 (현재 JS 110개 + Rust 17개).
- LICENSE 파일 추가.

### 수정 (v2.22)
- 업무 삭제 시 실행취소(토스트) 제공.
- 날짜/시간 오입력 거부(빨간 테두리 표시 + 저장 차단).
- 백업의 타임존 오류, torn-copy(불완전 복사) 방지, DB 무결성 게이트 수정.
