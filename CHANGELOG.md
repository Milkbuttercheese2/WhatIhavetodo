# 변경 이력 (CHANGELOG)

깃허브에 업데이트할 때마다 무엇이 바뀌었는지 이 파일에 기록한다.
버전 규칙: **v3.0.0부터 표준 X.Y.Z(semver)** — X=큰 개편, Y=기능 추가, Z=버그 수정.
(v2.31까지의 옛 규칙: 큰 업데이트 +0.1, 작은 업데이트 +0.01, 십진수 크기순 —
v2.2 → v2.21 → … → v2.3(=2.30) → v2.31. 즉 v2.3은 v2.23 다음(더 최신) 버전이다.)

## v3.1.0 — 2026-07-14

### 추가
- **주기 업무(반복)**: 양식 입력에 반복 선택(매주 요일 / 며칠마다 + 시각) 추가.
  반복을 켜면 마감시각이 자동 계산되고, 완료 체크 시 그 회차의 완료 기록이 [완료] 탭에
  남고(아카이빙) 다음 회차가 자동으로 다시 올라온다(세부 체크·알람 초기화, 실행취소 지원).
  카드에 '반복 · 매주 월·수 09:00' 태그 표시. items.recur 컬럼(마이그레이션 004, append-only) —
  기존 데이터·구버전 JSON 백업과 완전 호환.
- **빠른 메모창(Ctrl+Alt+Space) 개편**: Esc/포커스 이탈 시 내용을 절대 지우지 않음(삭제는
  사용자만). 작성 중 초안을 수시로 DB(settings.captureDraft)에 저장 — 앱/PC가 꺼져도 다음
  실행 때 초안이 '분류 대기'로 자동 등록(등록 시엔 초안 즉시 비워 중복 방지). **Alt 키로
  검색 모드 전환**: 왼쪽=내 업무(quick_search), 오른쪽=파일(Everything). 업무 클릭 → 메인
  창에서 열기, 파일 클릭 → 바로 열기. 창은 스포트라이트풍(투명 창 + 둥근 패널 + 그림자),
  메모 높이 2배(128px 상당).
- **설정 메뉴 통합**: 헤더의 저장 위치/JSON 백업/불러오기/XLSX/프리셋 관리를 [설정]
  드롭다운 하나로 통합(알람 토글만 밖에 유지). 메뉴에 체크 옵션 2종 — '실행 시 Everything
  자동 실행'(일반 설치 경로 자동 탐지, 없으면 조용히 생략), '빠른 검색 시 Everything 사용'.

### 변경
- UI 전반의 이모지 제거 (알람·저장 위치 버튼, 카드 파일 링크, 캘린더 등 — 텍스트로 대체).
- 마스코트 이미지는 이번 버전에서 보류 (원본 파일 미전달).

### 내부
- 새 모듈 recur.js(순수 반복 계산)·settings-menu.js·capture 검색 모드, Rust 커맨드
  quick_search/resize_capture/load_settings_only/launch_everything, 마이그레이션 004.
- 데이터 소실 집중 검증: recur 왕복·완료 재장전 상태 보존·LIKE 이스케이프·초안 게이트(F1)
  테스트 포함 — JS 101건 + Rust 21건 전부 통과.

## v3.0.0 — 2026-07-14

프로그램 이름이 **'뭐하려 했더라'** 로 바뀌고 버전 체계가 X.Y.Z로 전환된 큰 업데이트.
기존 데이터·JSON 백업·DB 백업과 완전 호환 (내부 식별자는 유지되어 데이터 경로가 끊기지 않음).

### 변경
- **프로그램 명칭 '뭐해야 했더라' → '뭐하려 했더라'**: 창 제목·헤더·알림·트레이 툴팁·
  백업 파일명·산출물 exe 파일명까지 일괄 변경. 새로 만들어지는 데이터 폴더 이름도
  `뭐하려했더라_데이터`로 변경 (기존 설치는 config.json의 절대경로를 그대로 사용 — 영향 없음).
- **버전 체계 X.Y.Z(semver) 전환**: 헤더의 버전 표기도 `v3.0.0` 그대로 표시 (옛 `.0` 절삭 폐지).
- **캘린더 표시 규칙 단순화**: 그날이 **마감일인 업무** + 그날 시각이 걸린 **세부 할일**만 표시.
  ("시각 있는 세부가 있으면 업무 마감일을 숨김" 로직 제거 — 이제 마감일은 항상 뜬다.)

### 추가
- **파일 링크**: 양식에서 업무에 파일 경로를 연결 ([＋ 파일 찾아보기] 또는 경로 붙여넣기).
  카드의 📄 클릭 = 파일 열기, 📂 = 폴더에서 보기. 검색창에서 파일명으로도 카드가 검색되고,
  XLSX 내보내기에 '파일링크' 열 추가. DB에 item_files 테이블 신설(마이그레이션 003),
  JSON 백업에 items[].files로 포함 (구버전 백업은 자동으로 빈 값 처리 — 왕복 호환).
- **Everything(voidtools) 파일 동시검색 (선택 기능)**: Everything의 HTTP 서버 옵션이 켜져
  있으면 보드 검색 시 내 PC 파일도 함께 검색되어 검색창 아래 표시, 클릭으로 바로 열기.
  통신은 127.0.0.1 로컬 한정. Everything 미실행 시 UI가 아예 나타나지 않는 무침습 설계
  (연결 실패 후 2분간 재시도 안 함). 사용법은 `사용 설명서.md` §5 참고.

### 수정
- **드래그 순서변경 미작동 수정**: 세부 할일·프리셋·식별번호 명칭의 ⠿ 드래그가 Windows에서
  아예 시작되지 않던 문제. 원인은 JS가 아니라 Tauri 창 설정 — WebView2에서 기본값
  `dragDropEnabled:true`가 OS 파일드롭을 위해 드래그 이벤트를 가로채 HTML5 드래그를 차단.
  `dragDropEnabled:false`로 해결 (트레이드오프: 창으로의 OS 파일 끌어놓기는 사용 불가 —
  이 앱은 원래 안 씀).
- **세부 할일 전부 완료 시 '예정 · 대기'로 떨어지던 배치 수정**: 이제 세부를 하나라도
  완료한 업무는 전부 완료했어도 (상위 완료 체크 전까지) '진행 중'에 남는다 —
  '예정 · 대기'는 손 안 댄 업무 전용.

### 내부
- 프론트 모듈 `everything.js` 신설, Rust 커맨드 `pick_file_path`/`open_file_path`/
  `reveal_file_path`/`everything_search` 추가 (Everything HTTP 클라이언트는 가짜 로컬
  서버 대상 단위테스트 4건 포함). 테스트: JS 91건 + Rust 21건 전부 통과.
- `기능 제안서.md`에 v3.0.0 기준 후속 제안(§8: 완료 아카이브 월별 묶음, 기간 XLSX,
  주간 리뷰 등) 추가. `사용 설명서.md`·`README.md`·`CLAUDE.md` 갱신.
- 산출물 exe는 다음 GitHub Actions(build-windows-exe) 실행 시 새 이름·새 기능으로 갱신됨
  (현재 커밋된 `뭐하려 했더라.exe`는 파일명만 바꾼 v2.31 빌드).

## v2.31 — 2026-07-11

툴바·기능을 덜어내는 정리 업데이트 — "단축키 → 메모창 → Ctrl+Enter 등록" 흐름만 남기고
주변 부가 기능(정기함·단축키 설정·상주 토글·저장 실패 표시)을 제거했다.

### 변경
- **미니 캡처(메모창) 단축키를 Ctrl+Alt+Space 고정으로**: 단축키 변경 UI(⚡ 빠른 메모
  버튼·설정 모달)를 삭제. 다른 프로그램 사용 중 단축키를 누르면 메모창이 뜨고,
  입력 후 Ctrl+Enter로 '분류 대기'에 등록되는 동작 자체는 그대로 유지.
- 트레이 메뉴를 '열기 / 종료' 둘로 정리 ('빠른 메모' 항목 삭제 — 메모창은 단축키로만).
- 창 X = 트레이로 숨김 동작은 유지 (토글 UI만 제거, 기본값 켬).

### 삭제
- **정기함(반복 일정) 기능 제거**: 🔁 정기함 버튼·모달·생성기(reconcileRecur)·카드 🔁 배지
  일괄 삭제. 이미 보드에 올라온 회차는 일반 카드라 그대로 남는다. DB의 recur_defs
  테이블과 JSON 백업의 recurDefs 항목은 **데이터 보존을 위해 왕복만 유지**(스키마
  마이그레이션은 append-only 원칙상 되돌리지 않음) — 기능만 사라졌고 데이터는 유실되지 않음.
- 헤더의 '⚠ 저장 실패' 상태 표시(saveStatus/setStatus) 제거 — 초기 로드 실패 시의
  안내(alert + 저장 차단)는 그대로 유지.
- 설정 모달에 있던 '윈도우 시작 시 자동 실행' 토글 제거 (기존에 켜둔 PC는 그대로 동작).

### 내부
- 프론트 모듈 `recur-box.js` 삭제, `capture-bridge.js`는 메모 라우팅+트레이 안내만 남김.
- Rust 커맨드 `save_recur_defs`/`set_capture_shortcut`/`set_autostart` 제거
  (백업 복원 경로의 recur_defs 저장은 유지).
- 테스트 JS 110 → 88개 (제거 기능의 테스트 삭제), Rust 17개 유지. `cargo check` 경고 0.
- 버전 v2.31 (매니페스트 2.31.0 ×3 + 락파일).

### 산출물
- `최종 프로그램 산출물/뭐해야 했더라.exe`를 v2.31 빌드로 갱신 (build-windows-exe
  워크플로 run #3 — windows-latest MSVC, 테스트 JS 88·Rust 17 통과 후 빌드, 단일 exe 12.5MB).

## v2.3 — 2026-07-11

v2.2 이후의 작은 중간 업데이트(v2.21, v2.22, v2.23)를 묶어 정식 반영한 **큰 업데이트**.
(v2.23의 다음 버전이 v2.3이다 — 위 버전 순서 규칙 참고.)

### 변경
- **보드 정렬 통일**: 세부할일(점검 시각)과 할일(마감시각)의 레벨 구분을 없애고,
  둘 중 먼저 도래하는 시각 순(오름차순)으로 카드를 정렬. 시각이 없는 카드는 뒤로,
  그 안에서는 최신 등록 순.
- **시각 도래 빨간 카드 제거**: 마감이 지났거나 임박했을 때 카드 전체 배경을
  빨갛게 칠하던 표시(urgent)를 삭제. 마감 태그(pill)의 빨간 지남/임박 표시는 유지.
- 프론트엔드 구조 개편(v2.21): 단일 `app.js`를 관심사별 ES 모듈로 분리
  (`state/store/render/form/placement/datetime/calendar/alarms/backup/presets/dom-utils/main`).
- 도메인 봉합선 정리: 아이템 생성을 `makeItem()` 팩토리로, 완료 토글을 `toggleDone()`
  도메인 연산으로 통일하고, 검색 술어를 `filters.js`로 분리 (동작 변화 없음 —
  반복 일정 등 후속 기능이 얹힐 기반).

### 추가
- **정기함(반복 일정)**: 매일/매주(요일 지정)/매월 반복 업무를 정기함(🔁)에 등록하면
  도래한 날에 일반 카드로 보드에 자동 등장(스폰). 반복 규칙은 아이템 속성이 아니라
  보드 밖 생성기(recur_defs)로 분리 — 체크=완료로 떠나는 보드 규칙과 `placeOf()` 불변.
  DB 마이그레이션 002(recur_defs 테이블 + items.recur_id), JSON 백업 자동 왕복.
- **전역 단축키 미니 캡처 창**(v2.23): 다른 프로그램 사용 중에도 단축키로 빠른 메모 입력.
- **트레이 상주 + 자동 시작**(v2.23): 창을 닫아도 트레이에 남고, 윈도우 시작 시 자동 실행 옵션.
- JS 테스트 하네스: node:test + jsdom 기반 자동 테스트 (현재 JS 110개 + Rust 17개).
- LICENSE 파일 추가.

### 수정 (v2.22)
- 업무 삭제 시 실행취소(토스트) 제공.
- 날짜/시간 오입력 거부(빨간 테두리 표시 + 저장 차단).
- 백업의 타임존 오류, torn-copy(불완전 복사) 방지, DB 무결성 게이트 수정.

### 기록
- 협업 워크플로 문서화(브랜치+PR 규칙), 시장 리서치 보고서 추가.
- `CHANGELOG.md` 도입 — 이 버전부터 매 업데이트의 변경 내역을 기록.
- `기능 제안서.md` 추가 — 조달청 내부망 관점 기능 추가 vs 단순 유지 제안 (보고서로만 유지, 신규 기능 개발은 보류).
- `사용 설명서.md` 추가 — 사용자 관점의 친절한 안내서 (시작하기·기능별 사용법·FAQ).
- README를 v2.3 현재 상태로 정비(전역 캡처·트레이 기능, 모듈 구조, 문서 목록).
- GitHub Actions 수동 빌드 워크플로 추가(`build-windows-exe.yml`) — Actions 탭에서
  버튼 한 번으로 Windows 러너가 정식 MSVC 빌드를 돌려 exe를 브랜치에 커밋.
  push/PR에는 반응하지 않는 수동 전용 (상시 CI 아님).

### 산출물
- `최종 프로그램 산출물/뭐해야 했더라.exe`를 **v2.3 빌드로 갱신 완료** (GitHub Actions
  windows-latest, MSVC 정적 링크 단일 exe 12.6MB, 테스트 JS 110·Rust 17 통과 후 빌드).
- 이후 갱신 방법 두 가지:
  ① GitHub **Actions 탭 → build-windows-exe → Run workflow** (Windows 러너가 MSVC 빌드 후 exe를 브랜치에 자동 커밋),
  ② Windows PC에서 `npm run tauri build` 후 `src-tauri/target/release/wmhh-desktop.exe`를 `뭐해야 했더라.exe`로 개명해 교체.
  (리눅스 MinGW 크로스빌드는 WebView2Loader.dll 동적 의존이 생겨 단일 exe 원칙에 맞지 않아 불채택.
  MSVC 정적 링크가 되는 Windows 네이티브 빌드만 공식 산출물로 사용.)
